# S5-01: Chat UI — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a browser chat interface where visitors can talk to the digital twin with real-time SSE streaming.

**Architecture:** AI SDK `useChat` hook with a custom `ProxyMindTransport` that bridges our FastAPI SSE format (`meta/token/citations/done`) to AI SDK's UI Message Stream. Session persistence via localStorage + backend API. shadcn/ui components with Tailwind CSS.

**Tech Stack:** React 19, Vite 8, Bun, TypeScript, AI SDK (`ai` + `@ai-sdk/react`), React Router, shadcn/ui, Tailwind CSS, react-markdown, rehype-sanitize, Vitest, React Testing Library.

**Spec:** `docs/superpowers/specs/2026-03-25-s5-01-chat-ui-design.md`

- **Commits:** NEVER commit without explicit user permission. Commit steps in this plan are templates — always ask before executing.
- **Pre-implementation:** Read `docs/development.md` before writing code (Task 1, Step 0).
- **Post-implementation:** Re-read `docs/development.md` and self-review the change against it (Task 13).
- **Version policy:** Verify all installed package versions are not below minimums in `docs/spec.md`.
- **CI:** Run CI tests and confirm they pass before considering the change complete.

**IMPORTANT:** Before writing the transport layer (Task 4), the implementer MUST verify the current AI SDK v6 `@ai-sdk/react` transport/`useChat` API from official docs. If `ChatTransport` or the `fetch` option cannot accommodate our custom SSE format, fall back to Approach B (custom hook with AI SDK streaming utilities only). Task 4 (transport) comes before Task 5 (message adapter) because the adapter's output format depends on the chosen integration approach.

**Approach A/B ripple effect:** The choice in Task 4 affects more than just `transport.ts`:
- **Approach A (`useChat`):** Messages are `UIMessage[]` with `.parts` array (not flat `.content`). Components (Tasks 8, 10, 11) must extract text via `message.parts.filter(p => p.type === 'text')`. Chat status uses AI SDK's enum: `submitted | streaming | ready | error`. Task 5 adapter outputs `UIMessage[]`. Initial history is passed via `messages` prop (verify exact param name — AI SDK v6 may use `messages` not `initialMessages`). Disable state must check `status === "submitted" || status === "streaming"`.
- **Approach B (custom hook):** Messages use our flat `ChatMessage` type with `.content` string. Components read `message.content` directly. Custom status enum: `idle | streaming | error`. Task 5 adapter outputs `ChatMessage[]`. Disable state checks `status === "streaming"`.

**The plan below provides concrete code for Approach B's `ChatMessage` format.** This is deliberate — Approach B code is self-contained and testable without external API verification. If Approach A is chosen in Task 4, the implementer MUST adapt:
- Task 5: adapter output → `UIMessage[]` with `.parts`, not `ChatMessage[]`
- Task 6: `useSession` returns adapted `UIMessage[]` as initial state
- Task 8: `MessageBubble` reads `message.parts` not `message.content`
- Task 10: `MessageList` passes `UIMessage[]` not `ChatMessage[]`
- Task 11: `ChatPageInner` passes history via `messages` param (verify name); disable condition includes `submitted`

```typescript
import type { Session, SessionWithMessages } from "../types/chat";

export async function createSession(apiUrl: string): Promise<Session> {
  const response = await fetch(`${apiUrl}/api/chat/sessions`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
  });

  if (!response.ok) {
    throw new Error(`Failed to create session: ${response.status}`);
  }

  return response.json();
}

export async function getSession(
  apiUrl: string,
  sessionId: string,
): Promise<SessionWithMessages | null> {
  const response = await fetch(`${apiUrl}/api/chat/sessions/${sessionId}`);

  if (response.status === 404) {
    return null;
  }

  if (!response.ok) {
    throw new Error(`Failed to get session: ${response.status}`);
  }

  return response.json();
}
```

```typescript
import { describe, it, expect } from "vitest";
import { parseSSEStream } from "../../lib/sse-parser";
import type { SSEEvent } from "../../types/chat";

function createStream(text: string): ReadableStream<Uint8Array> {
  const encoder = new TextEncoder();
  return new ReadableStream({
    start(controller) {
      controller.enqueue(encoder.encode(text));
      controller.close();
    },
  });
}

function createChunkedStream(chunks: string[]): ReadableStream<Uint8Array> {
  const encoder = new TextEncoder();
  return new ReadableStream({
    start(controller) {
      for (const chunk of chunks) {
        controller.enqueue(encoder.encode(chunk));
      }
      controller.close();
    },
  });
}

async function collectEvents(stream: ReadableStream<Uint8Array>): Promise<SSEEvent[]> {
  const events: SSEEvent[] = [];
  for await (const event of parseSSEStream(stream)) {
    events.push(event);
  }
  return events;
}

describe("parseSSEStream", () => {
  it("parses a single token event", async () => {
    const stream = createStream('event: token\ndata: {"content":"Hello"}\n\n');
    const events = await collectEvents(stream);

    expect(events[0]).toEqual({ type: "token", content: "Hello" });
```

```typescript
/**
 * Parses a ReadableStream of SSE data into typed events.
 * Handles partial chunks, heartbeat comments, and all backend event types.
 */
export async function* parseSSEStream(
  stream: ReadableStream<Uint8Array>,
): AsyncGenerator<SSEEvent> {
  const reader = stream.getReader();
  const decoder = new TextDecoder();
  let buffer = "";

  try {
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;

      buffer += decoder.decode(value, { stream: true });

      // SSE events are separated by double newlines
      const parts = buffer.split("\n\n");
      // Last part may be incomplete — keep in buffer
      buffer = parts.pop() ?? "";

      for (const part of parts) {
        const trimmed = part.trim();
        if (!trimmed || trimmed.startsWith(":")) {
          // Empty or comment (heartbeat) — skip
          continue;
        }

        const event = parseSSEBlock(trimmed);
        if (event) {
          yield event;
        }
      }
    }
  } finally {
    reader.releaseLock();
  }
}

function parseSSEBlock(block: string): SSEEvent | null {
  let eventType = "";
  let dataStr = "";

  for (const line of block.split("\n")) {
    if (line.startsWith("event: ")) {
      eventType = line.slice(7).trim();
    } else if (line.startsWith("data: ")) {
      dataStr = line.slice(6);
    }
  }

  if (!eventType || !dataStr) return null;

  try {
    const data = JSON.parse(dataStr);

    switch (eventType) {
      case "meta":
        return {
          type: "meta",
          message_id: data.message_id,
          session_id: data.session_id,
          snapshot_id: data.snapshot_id,
        };
      case "token":
        return { type: "token", content: data.content };
      case "citations":
        return { type: "citations", citations: data.citations };
      case "done":
        return {
          type: "done",
          token_count_prompt: data.token_count_prompt,
          token_count_completion: data.token_count_completion,
          model_name: data.model_name,
          retrieved_chunks_count: data.retrieved_chunks_count,
        };
      case "error":
        return { type: "error", detail: data.detail };
      default:
        return null;
    }
  } catch {
    return null;
  }
}
```

```typescript
export function toUIMessages(messages: MessageInHistory[]): ChatMessage[] {

    case "complete":
      return "complete";
    case "partial":
      return "partial";
    case "failed":
      return "failed";
  }
}
```

```typescript
import { useCallback, useEffect, useState } from "react";
import type { ChatMessage } from "../types/chat";
import { createSession, getSession } from "../lib/api";
import { config, STORAGE_KEY_SESSION } from "../lib/config";
import { toUIMessages } from "../lib/message-adapter";

export function useSession() {
  const [sessionId, setSessionId] = useState<string | null>(null);
  const [initialMessages, setInitialMessages] = useState<ChatMessage[]>([]);
  const [isReady, setIsReady] = useState(false);

  useEffect(() => {
    let cancelled = false;

    async function init() {
      const storedId = localStorage.getItem(STORAGE_KEY_SESSION);

      if (storedId) {
        const session = await getSession(config.apiUrl, storedId);
        if (!cancelled && session) {
          setSessionId(storedId);
          setInitialMessages(toUIMessages(session.messages));
          setIsReady(true);
          return;
        }
        // Session not found — fall through to create new
      }

      const newSession = await createSession(config.apiUrl);
      if (!cancelled) {
        localStorage.setItem(STORAGE_KEY_SESSION, newSession.id);
        setSessionId(newSession.id);
        setInitialMessages([]);
        setIsReady(true);
      }
    }

    init();
    return () => { cancelled = true; };
  }, []);

  const createNewSession = useCallback(async () => {
    const newSession = await createSession(config.apiUrl);
    localStorage.setItem(STORAGE_KEY_SESSION, newSession.id);
    setSessionId(newSession.id);
    setInitialMessages([]);
  }, []);

  return { sessionId, initialMessages, isReady, createNewSession };
}
```
